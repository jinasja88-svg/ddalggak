# 1688 소싱 플랫폼 구축 계획

> 소싱라이프 유사 서비스 — 기존 딸깍러(Next.js 16 + Supabase) 스택 기반 확장

---

## 1. 전체 아키텍처

```
사용자 브라우저
    ↓ (AJAX / React Query)
Next.js API Routes (프록시 레이어)
    ↓ (서버 ↔ 서버, API Key 숨김)
1688 API (서드파티 or 공식)
    ↓
Supabase (캐싱 + 주문 저장)
    ↓ 캐시 히트 시
Next.js → 브라우저 즉시 응답
```

**핵심 원칙**: 브라우저에서 1688 API를 직접 호출하지 않음. 반드시 Next.js 서버를 통해 프록시.

## 2. 1688 API 접근 방법 비교

| 방법 | 난이도 | 비용 | 현실성 | 추천 |
|------|--------|------|--------|------|
| **공식 알리바바 Open Platform** | 매우 높음 | 낮음 | 낮음 (중국 사업자 필요, 심사 수개월) | ❌ |
| **OTCommerce API** | 낮음 | $99~$299/월 | 높음 (한국어 지원, RESTful) | ✅ **1순위** |
| **TMAPI** | 낮음 | 종량제 | 높음 (1회 호출당 과금) | ✅ **2순위** |
| **직접 크롤링** | 중간 | 서버 비용만 | 낮음 (IP 차단, ToS 위반 위험) | ❌ |

### 추천: OTCommerce API
- 엔드포인트: `https://api.otcommerce.com/`
- 기능: 상품 검색, 상세, 이미지 검색, 주문, 배송 추적
- 한국어 번역 자동 지원
- 가격: [otcommerce.com 요금제 확인](https://otcommerce.com/pricing)

### 대안: TMAPI
- 엔드포인트: `https://api.tmapi.top/`
- 종량제 (1,000회 호출당 $X)
- 문서: [tmapi.top/docs](https://tmapi.top/docs)

## 5. DB 스키마 확장 (Supabase)

기존 `supabase/schema.sql`에 아래 테이블 추가:

```sql
-- 1688 상품 캐시
CREATE TABLE sourcing_products (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  product_id TEXT UNIQUE NOT NULL,        -- 1688 상품 ID
  title TEXT NOT NULL,                     -- 한국어 번역 제목
  title_zh TEXT,                           -- 중국어 원제목
  price_cny DECIMAL(10,2),                 -- 위안 가격
  price_krw INTEGER,                       -- 원화 환산 (캐싱 시점)
  images JSONB,                            -- 이미지 URL 배열
  skus JSONB,                              -- 옵션(색상, 사이즈 등)
  seller_id TEXT,
  seller_name TEXT,
  category_id TEXT,
  stock INTEGER,
  raw_data JSONB,                          -- 원본 API 응답 전체
  cached_at TIMESTAMPTZ DEFAULT now(),
  created_at TIMESTAMPTZ DEFAULT now()
);

-- 수입대행 주문
CREATE TABLE sourcing_orders (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id),
  order_number TEXT UNIQUE NOT NULL,      -- 내부 주문번호
  status TEXT DEFAULT 'pending',          -- pending|paid|purchasing|shipping|delivered|cancelled
  items JSONB NOT NULL,                   -- [{product_id, title, qty, price_cny, sku}]
  total_cny DECIMAL(10,2),
  total_krw INTEGER,
  service_fee INTEGER,                    -- 수수료
  shipping_fee INTEGER,                   -- 국내배송비
  paid_amount INTEGER,                    -- 실결제금액
  toss_payment_key TEXT,
  shipping_address JSONB,                 -- {name, phone, address, postal_code}
  tracking_number TEXT,                   -- 운송장 번호
  admin_note TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- 환율 캐시
CREATE TABLE exchange_rates (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  from_currency TEXT NOT NULL,            -- CNY
  to_currency TEXT NOT NULL,              -- KRW
  rate DECIMAL(10,4) NOT NULL,
  fetched_at TIMESTAMPTZ DEFAULT now()
);

-- RLS 설정
ALTER TABLE sourcing_products ENABLE ROW LEVEL SECURITY;
ALTER TABLE sourcing_orders ENABLE ROW LEVEL SECURITY;
ALTER TABLE exchange_rates ENABLE ROW LEVEL SECURITY;

-- sourcing_orders: 본인 주문만 조회
CREATE POLICY "Users can view own sourcing orders" ON sourcing_orders
  FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Users can create sourcing orders" ON sourcing_orders
  FOR INSERT WITH CHECK (auth.uid() = user_id);

-- sourcing_products: 전체 공개
CREATE POLICY "Public read sourcing_products" ON sourcing_products
  FOR SELECT USING (true);

-- exchange_rates: 전체 공개
CREATE POLICY "Public read exchange_rates" ON exchange_rates
  FOR SELECT USING (true);
```

## 6. API 라우트 설계 (Next.js)

```
src/app/api/
├── sourcing/
│   ├── search/route.ts          # GET ?q=가방&page=1&category=xxx
│   ├── product/[id]/route.ts   # GET 상품 상세
│   ├── image-search/route.ts   # POST { imageData: base64 }
│   ├── categories/route.ts     # GET 카테고리 트리
│   └── orders/
│       ├── route.ts             # GET 목록 / POST 생성
│       └── [id]/route.ts        # GET 상세 / PATCH 상태변경
└── exchange-rate/route.ts       # GET 현재 환율 (CNY→KRW)
```

### 상품 검색 예시 (`/api/sourcing/search`)
```typescript
// 요청
GET /api/sourcing/search?q=가방&page=1

// 처리 흐름
1. 캐시 확인 (Redis/Supabase)
2. 캐시 미스 → OTCommerce API 호출
   POST https://api.otcommerce.com/offerlist/search
   { keyword: "包", page: 1, pageSize: 20 }
3. 응답 데이터 Supabase에 upsert (sourcing_products)
4. 환율 적용 → 원화 가격 계산
5. 브라우저에 반환

// 응답 형태
{
  products: [
    {
      id: "123456789",
      title: "여성 가죽 핸드백",
      price_cny: 45.0,
      price_krw: 83700,
      images: ["https://..."],
      seller: "상하이 패션몰"
    }
  ],

  page: 1
```

## 7. 가격 계산 로직

```
최종 한국 판매가 = (위안가 × 환율) + 중국내배송비 + 수입대행수수료 + 관세 + 국내배송비

예시:
- 상품가: ¥50 × 185원/위안 = 9,250원
- 중국내배송비: 약 2,000~5,000원 (무게별 상이)
- 수입대행 수수료: 상품가의 10~15%
- 관세: 목록통관 150달러 이하 면세 (소액면세)
- 국내배송비: 3,000원

→ 예상 총비용: 약 15,000~20,000원
```

**환율 API 연동**:
```typescript
// ExchangeRate-API (무료 플랜: 1,500 req/month)
GET https://v6.exchangerate-api.com/v6/{API_KEY}/latest/CNY
// → KRW 환율 반환, 1시간마다 Supabase exchange_rates 테이블 갱신
```

## 9. 수입대행 주문 플로우

```
1. 사용자: 상품 상세에서 수량/옵션 선택
2. 수입대행 신청서 작성 (배송지, 특이사항)
3. 견적 확인 (총금액 = 상품가 + 수수료 + 예상배송비)
4. Toss Payments 결제 (기존 /api/checkout/confirm 재활용)
5. DB: sourcing_orders 레코드 생성 (status: 'paid')
6. 관리자: 1688에서 실제 주문 진행 (수동 or API)
7. 관리자: 운송장 번호 입력 → status: 'shipping'
8. 사용자: 배송 추적 조회
9. 배송 완료 → status: 'delivered'
```
